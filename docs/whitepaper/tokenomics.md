## EXECUTIVE SUMMARY / 执行摘要

### English

$POI is the native utility token for the ProjectEX platform, aligning Users, Creators, Brands, and Investors. The system uses a simple, auditable monetary policy (mining-based issuance plus adaptive burns) and verifiable distributions (staking + influence mining). Objectives: predictable long-term value capture and net inflation ≤ 5% (annualized), while maximizing retention and growth.

**Core Goals:**

- Bound net inflation via adaptive burn (≤ 5% annualized target).
- Reward long-term participation: staking, quality creation, genuine engagement.
- Transparent daily vesting and defensible market-opening mechanics.
- Compliance-aware Web2 → Web3 migration without user friction.

### 中文

$POI 是 ProjectEX 平台的原生实用代币，协调用户、创作者、品牌和投资者。该系统采用简单、可审计的货币政策（基于挖矿的发行加上自适应燃烧）和可验证的分配（质押 + 影响力挖矿）。目标：可预测的长期价值捕获和净通胀率 ≤ 5%（年化），同时最大化留存和增长。

**核心目标：**

- 通过自适应燃烧限制净通胀（≤ 5% 年化目标）。
- 奖励长期参与：质押、优质创作、真实参与。
- 透明的每日解锁和可靠的市场开放机制。
- 合规的 Web2 → Web3 迁移，无用户摩擦。

## MARKET OPENING AND LIQUIDITY / 市场开放与流动性

### Protocol-Owned Liquidity / 协议拥有的流动性

Protocol-Owned Liquidity (LP allocation) deploys only after trading is enabled.

- **Initial Injection** (Listing Day): 20-30% of LP bucket.
- **Tranche Add-ons** (Phases 1-2): remaining LP added in multiple tranches via volume/volatility triggers (e.g., add when 24h volume > X and intraday volatility < Y).
- **Buyback & Re-deployment**: if 7D VWAP is below longer-term average by threshold, treasury may buy back for future MM or burns (subject to governance).
- **Abuse Protections**: per-tx caps on Day 1, progressive slippage guard, multisig + timelocks for treasury.

### 中文

协议拥有的流动性（LP 分配）仅在交易启用后部署。

- **初始注入**（上市日）：LP 池的 20-30%。
- **分批增补**（阶段 1-2）：通过交易量/波动性触发机制分批添加剩余 LP（例如，当 24 小时交易量 > X 且日内波动性 < Y 时添加）。
- **回购与重新部署**：如果 7 日 VWAP 低于长期平均值超过阈值，金库可回购用于未来做市或燃烧（需治理批准）。
- **滥用保护**：首日每笔交易上限、渐进滑点保护、金库多签 + 时间锁。

## MONETARY POLICY: ISSUANCE AND BURNS / 货币政策：发行与燃烧

### 6.1 Staking Mining / 质押挖矿

- **Baseline Daily Emission**: E_day,baseline ≈ Pool Allocation / Vesting Period (governance may apply decay).
- **APY Target Band**: 8%-25% to be attractive yet sustainable.
- r_day = (E_day + F_day) / S_total; APY ≈ r_day × 365.
- If APY > upper band → reduce E_day; if APY < lower band → increase E_day and/or fee-share.
- **Long-Stake Multipliers**: commitments of 30/90/180 days → 1.1× / 1.2× / 1.3× weight.
- **Early Unstake Penalty**: sliding fee; partially burned.

### 中文

- **基准每日排放**：E_day,baseline ≈ 池分配 / 解锁期（治理可应用衰减）。
- **APY 目标区间**：8%-25%，既具吸引力又可持续。
- r_day = (E_day + F_day) / S_total; APY ≈ r_day × 365。
- 如果 APY > 上限 → 减少 E_day；如果 APY < 下限 → 增加 E_day 和/或费用分成。
- **长期质押倍数**：30/90/180 天承诺 → 1.1× / 1.2× / 1.3× 权重。
- **提前解质押 penalty**：滑动费用；部分燃烧。

### 6.2 Influence Mining / 影响力挖矿

- **Daily Emission**: equal division over vesting period (governance may apply decay).
- **Influence Score** (creator i):

```
IS_i = w1·PaidFans_i + w2·sqrt(ActiveFans_i) + w3·Retention_i + 
       w4·GMV_norm_i + w5·Onchain_i + w6·Compliance_i - Penalty_i
```

- **Daily Payout**: P_i = E_day × (IS_i / ΣIS). Optional: 50% immediate + 50% linear over 90 days to reduce sell pressure.
- **Fan Quests**: small sub-pool for genuine fan actions (referrals, verifications, streaks).

### 中文

- **每日排放**：在解锁期内平均分配（治理可应用衰减）。
- **影响力评分**（创作者 i）：

```
IS_i = w1·付费粉丝_i + w2·sqrt(活跃粉丝_i) + w3·留存率_i + 
       w4·GMV标准化_i + w5·链上_i + w6·合规_i - 惩罚_i
```

- **每日支付**：P_i = E_day × (IS_i / ΣIS)。可选：50% 立即 + 50% 90 天线性解锁，以减少抛售压力。
- **粉丝任务**：为真实粉丝行为（推荐、验证、连续参与）提供的小型子池。

### 6.3 Burn Rules / 燃烧规则

**Low-take** (Web2/C2C/fiat & low-friction flows):

- C2C resale: burn 0.5% of GMV.
- Fiat top-up: burn 1.0% of top-up.
- Subscriptions: burn 1.0-2.0% of payment.
- In-app transfers/gifts: burn 0.2% (daily cap per account).
- In-app trades: burn 0.5-1.0% of executed value.

**High-burn** (Web3/exit/fees/DeFi):

- Off-ramp withdrawals (POI → Fiat/USDC): burn 3-8% (tiered by KYC/frequency/size).
- POI-denominated payments: burn 2-5%.
- Tips: burn 3%.
- KYC/fee items: burn 20-100% of those fee tokens.
- DeFi actions (stake/unstake/bridge/borrow): burn 1-3% of the relevant amount.

### 中文

**低费率**（Web2/C2C/法币和低摩擦流程）：

- C2C 转售：燃烧 GMV 的 0.5%。
- 法币充值：燃烧充值的 1.0%。
- 订阅：燃烧支付的 1.0-2.0%。
- 应用内转账/礼物：燃烧 0.2%（每个账户每日上限）。
- 应用内交易：燃烧执行价值的 0.5-1.0%。

**高燃烧**（Web3/退出/费用/DeFi）：

- 法币提现（POI → 法币/USDC）：燃烧 3-8%（按 KYC/频率/规模分级）。
- POI 计价支付：燃烧 2-5%。
- 打赏：燃烧 3%。
- KYC/费用项目：燃烧这些费用代币的 20-100%。
- DeFi 操作（质押/解质押/桥接/借贷）：燃烧相关金额的 1-3%。

### 6.4 Adaptive Burn Controller (Net Inflation Guard) / 自适应燃烧控制器（净通胀保护）

- NetInflation_30D = (Mint_30D - Burn_30D) / Avg Circulating_30D.
- If annualized NetInflation_30D > 5%: multiply all burn rates by (1+k), e.g., k = 10-30%.
- If annualized NetInflation_30D < 1%: consider reducing burn and/or increasing emissions or staker fee-share.
- Review: weekly; parameters on-chain and governance-controlled.

### 中文

- NetInflation_30D = (Mint_30D - Burn_30D) / Avg Circulating_30D。
- 如果年化 NetInflation_30D > 5%：将所有燃烧率乘以 (1+k)，例如 k = 10-30%。
- 如果年化 NetInflation_30D < 1%：考虑减少燃烧和/或增加排放或质押者费用分成。
- 审查：每周；参数在链上且由治理控制。

## DISTRIBUTION, DATA INTEGRITY, AND ANTI-ABUSE / 分配、数据完整性和反滥用

- **Web2 events**: Event Gateway with device fingerprinting, behavior filters, anomaly checks; hashed receipts anchored on-chain.
- **Web3 events**: natively on-chain (subs, tips, NFT consumptions).
- **TGE → Phase 1**: spend-only balances (no transfers), enabling growth loops without speculative volatility; balances become transferable when trading opens.
- **Risk Controls**: high-risk throttles, cool-downs, sybil scoring, reward blacklists.

### 中文

- **Web2 事件**：事件网关，包含设备指纹识别、行为过滤器、异常检查；哈希收据锚定在链上。
- **Web3 事件**：原生链上（订阅、打赏、NFT 消费）。
- **TGE → 阶段 1**：仅消费余额（不可转账），在没有投机波动的情况下实现增长循环；交易开放时余额变为可转账。
- **风险控制**：高风险节流、冷却期、女巫评分、奖励黑名单。

## KEY RISKS AND MITIGATIONS / 主要风险与缓解措施

- **Market Volatility**: LP tranches and buyback rules; adaptive burn bounds net inflation.
- **Sybil/Farming Abuse**: device/graph analysis, velocity limits, delayed claims, penalty slashing.
- **Ill-targeted Emissions**: APY band targeting for staking; influence scoring with penalties/audits.
- **Regulatory Shifts**: modular compliance, KYC gating on exits, geo-fencing as needed.
- **Smart Contract Risk**: audits, bug bounties, pause/guarded launches.

### 中文

- **市场波动**：LP 分批和回购规则；自适应燃烧限制净通胀。
- **女巫/刷量滥用**：设备/图分析、速度限制、延迟申领、惩罚削减。
- **目标不当的排放**：质押的 APY 区间目标；带惩罚/审计的影响力评分。
- **监管变化**：模块化合规、退出时的 KYC 门控、根据需要的地理围栏。
- **智能合约风险**：审计、漏洞赏金、暂停/保护性启动。
