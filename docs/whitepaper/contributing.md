# Contributing to Acee

**How to Contribute to the Value Internet**

---

## Introduction / 介绍

### English

Acee is building the Value Internet — a unified infrastructure for standardizing, verifying, and coordinating value across the digital economy. We welcome contributions from developers, creators, brands, researchers, and community members who share our vision.

This guide outlines how you can contribute to Acee's ecosystem through code, documentation, community participation, and partnerships.

### 中文

Acee 正在构建价值互联网——一个统一的基础设施，用于标准化、验证和协调数字经济中的价值。我们欢迎开发者、创作者、品牌、研究人员和社区成员的贡献，他们与我们分享相同的愿景。

## Contribution Process / 贡献流程

### For Code Contributions / 代码贡献

1. **Fork & Clone**: Fork the repository and clone to your local machine
2. **Create Branch**: Create a feature branch from `dev` (or `main` for hotfixes)
3. **Make Changes**: Implement your feature or fix
4. **Test**: Ensure all tests pass and add new tests if needed
5. **Document**: Update relevant documentation
6. **Commit**: Write clear commit messages following conventions
7. **Push & PR**: Push to your fork and create a pull request
8. **Review**: Address feedback and iterate
9. **Merge**: Once approved, your changes will be merged

### For Documentation / 文档

1. **Identify Need**: Find areas that need documentation
2. **Create/Edit**: Write or update documentation
3. **Review**: Self-review for clarity and accuracy
4. **Submit**: Create a pull request or issue
5. **Iterate**: Address feedback and improve

### For Community / 社区

1. **Engage**: Participate in Discord, social media, or events
2. **Help**: Answer questions, provide guidance
3. **Share**: Create content, tutorials, or resources
4. **Organize**: Host events or workshops
5. **Feedback**: Provide constructive feedback and suggestions

## Coding Standards / 编码标准

### TypeScript / TypeScript

- Use strict mode
- Prefer interfaces over types for props
- Avoid `any`, use `unknown` when needed
- Shared types in `shared/schema.ts`

### React Components / React 组件

- Functional components with hooks
- Props destructuring with type annotations
- `data-testid` attributes for testing
- Event handlers: `handle[EventName]`

### Smart Contracts / 智能合约

- Solidity 0.8.20
- OpenZeppelin contracts
- Security best practices
- Comprehensive testing

### Git Commit Messages / Git 提交消息

Format: `<type>(<scope>): <description> (Cursor)`

Types:
- `feat`: New feature
- `fix`: Bug fix
- `refactor`: Code refactoring
- `style`: Style changes
- `docs`: Documentation
- `chore`: Build/tools
